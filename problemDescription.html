Pop a Red Balloon

The main goal of this project is to develop and implement the code required for the drone (UAV) or (UGV) to automatically find and pop a red balloon. Or some other relatively easy to identify, but which must first be visually found.
To this end, a video camera is connected to the drone and video processing is used to first detect the red balloon, and then to navigate the drone in the right direction to pop the balloon with the propellers of the drone (or a poker if a UGV is used).
The location of the target will change as the drone moves, but this can be compensated by receiving IMU information (pitch, roll), either from the autopilot, or from a separate IMU.
This project is closely related to visual tracker project and the video selfie project.

Current Status

In Spring 2017, a team attempted this project. The results were underwhelming (although they did pop a balloon with the propellers), but the entire process of recognizing the balloon was very slow and unreliable, and they had a host of problems from stabilizing the video stream, to controlling the yaw of the UAV, to being unable to use a laser pointer to pop the balloon (as they hoped to). Also the library/processor they used only allowed for ... 1 frame per second processing or so, which had to slow down everything.

Requirements

To successfully complete this project you need to have (or acquire):
  • Ability to fly a drone if UAV.
  • Programming skills to implement any algorithm you come up with.
  • Basic video processing.
